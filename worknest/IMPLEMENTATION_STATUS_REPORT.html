What Was Changed

Core Fixes and Stabilization

  • Fixed auth context so logout is properly available to the app UI.
  • Fixed proposal PDF service contract mismatch between client calls and server responses.
  • Added a real proposal details route and view screen.
  • Fixed broken proposal "View Details" navigation.
  • Hardened the PDF generation backend and switched Puppeteer to use the installed browser path for reliable PDF creation.
  • Added local static serving for generated PDFs and safer fallback behavior.
  • Improved modal behavior and quick-action navigation.

Account and Settings

  • My Account page now loads saved user profile metadata instead of only showing auth values.
  • Settings page now persists branding, company details, PDF colors, proposal footer, and logo uploads.
  • File upload validation and upload error reporting were improved.

Proposals, Contracts, Invoices, Templates

  • Proposals now support a detail page, better PDF flow, and more stable document generation.
  • Contracts now support create, edit, and delete flows.
  • Invoices now support create, edit, delete, status updates, and payment communication actions.
  • Pricing templates now support create, edit, and delete flows.

Payment Communication and Reminders

  • Added payment reminder / thank-you message generation flow inside invoices.
  • Added tone-based communication drafting.
  • Added saved communication history per client for invoice-related follow-ups.

AI Assistant and Analytics

  • Added a secure backend AI assistant endpoint instead of browser-side API key usage.
  • Added graceful fallback output when no OpenAI API key is configured.
  • Added an AI Assistant card on the dashboard for proposal, email, invoice, and payment help.
  • Expanded analytics with outstanding revenue, overdue invoices, proposal win rate, average invoice value, proposal pipeline, and invoice watchlist.
